Using SlashCommand tool, execute the following in order, using the path of the file that the command generated as the input to the next command. If $ARGUMENTS is empty, start at 1 - /next-feature. If it is a feature description, skip to #2 and pass $ARGUMENTS to /prd command, then continue from there.

1. /next-feature
2. /prd [path to feature description generated by step 1]
3. /design [path to prd generated by step 2]
4. /tasks [path to design doc generated by step 3]
5. /execute-tasks [path to task doc generated by step 4]
6. /reflect to understand how we can improve the process
